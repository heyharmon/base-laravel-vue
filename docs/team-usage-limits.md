# Team Usage Limits and Monthly Usage Reporting

This document explains how team usage limits are enforced and how monthly usage is calculated and displayed across the backend (Laravel) and frontend (Vue + Pinia).

## Overview

- Monthly usage is tracked for each team across two sources:
  - Responses generated by prompts (`responses` table)
  - Chats generated in article conversations (`chats` table)
- Each record stores token usage metadata and two monetary figures:
  - `cost`: internal cost basis (from `config/llm.php`)
  - `price`: billable price basis (environment-configurable multipliers)
- Teams can have an optional monthly cost limit (`teams.token_limit_cost`).
  - When set to `null`, the team is unlimited.
  - When set to a number, new prompt runs and chats are blocked once the current month’s accumulated `cost` exceeds the limit.
- Super admins can view per-team monthly usage and remaining budget, navigate to a team’s detail page to see a breakdown by Responses and Chats, and update the monthly limit.

## Data Model

- Migration: `database/migrations/2025_08_14_000000_add_token_limits_to_teams_table.php`
  - Adds nullable decimals: `token_limit_cost`, `token_limit_price` to `teams`.
- Team model: `app/Models/Team.php`
  - `$fillable`: includes `token_limit_cost`, `token_limit_price`.
  - `$casts`: both cast to `float`.
- Response model: `app/Models/Response.php`
  - Fields include `usage` (JSON), `cost` (float), `price` (float).
- Chat model: `app/Models/Chat.php`
  - Fields include `usage` (JSON), `cost` (float), `price` (float).

## Cost and Price Configuration

- `config/llm.php` defines per‑token `cost` and `price` for OpenAI models (e.g., `gpt-4o`, `gpt-5`).
- Token calculators (within job/services that call LLMs) compute `cost` and `price` from the `usage` object for each response/chat and persist it with the record.

## Backend Services and Enforcement

### Service: `app/Services/TeamUsageService.php`

- `currentMonthCost(Team $team)`: sums `cost` for responses and chats for the current month.
- `currentMonthPrice(Team $team)`: sums `price` for the current month.
- `exceedsLimit(Team $team, float $additionalCost = 0)`: true if `currentMonthCost + additionalCost > team.token_limit_cost`; unlimited if the field is `null`.
- `ensureWithinLimit(Team $team, float $additionalCost = 0)`: throws `App\Exceptions\UsageLimitExceeded` if the limit would be exceeded.
- `usageForPeriod(Team $team, Carbon $start, Carbon $end)`: aggregates counts, tokens, and cost for Responses and Chats, plus totals, over an arbitrary period (used for monthly reporting).

### Enforcement Points

- Prompt runs: `app/Jobs/RunPromptJob.php`
  - Calls `TeamUsageService::ensureWithinLimit($team)` before requesting an LLM.
- Article chat API: `app/Http/Controllers/ArticleChatController.php`
  - Calls `ensureWithinLimit($article->team)` before sending a chat request.
- OpenAI service calls: `app/Services/OpenAIArticleAgentService.php`
  - Calls `ensureWithinLimit($team)` before creating responses (both sync and async paths).

Note: Enforcement checks the current accrued monthly `cost` before the new call. It doesn’t estimate the incremental cost of the pending call, so a single large call could push a team slightly over the limit. If you want stricter pre‑check behavior, pass an estimated `additionalCost` to `ensureWithinLimit` at the call sites.

## Backend API

### Team (member) usage summary

- Route: `GET /api/teams/{team}/usage`
- Controller: `app/Http/Controllers/TeamUsageController.php`
- Auth: must belong to the team; returns the current month’s `usage_cost`, `usage_price`, `limit_cost`, `limit_price`.

Response example:

```json
{
  "usage_cost": 12.34,
  "usage_price": 24.68,
  "limit_cost": 100.0,
  "limit_price": 200.0
}
```

### Super Admin usage list and detail

- List teams with current usage and remaining budget
  - `GET /api/super-admin/teams`
  - Returns: `[{ id, name, usage_cost, limit_cost, remaining_cost }]`

- Team monthly detail (with period selection)
  - `GET /api/super-admin/teams/{team}/usage?month=YYYY-MM`
  - Returns:
    - `team`: `{ id, name }`
    - `period`: `{ start, end, days_until_reset }`
    - `usage`: `{ responses: { count, tokens, cost }, chats: { count, tokens, cost }, total: { tokens, cost } }`
    - `limit_cost`: number|null

- Update team limit (super admin)
  - `PUT /api/super-admin/teams/{team}/limit`
  - Body: `{ token_limit_cost: number|null, token_limit_price?: number|null }`
  - Returns: updated team.

Access control: All super admin endpoints are protected by `App\Http\Middleware\SuperAdminMiddleware` and routed under `routes/api.php` via the `super_admin` middleware group.

Days until reset: For the current month, the controller computes days until the next monthly reset. For past months, this is returned as `0`.

## Frontend

### API Wrapper

- `resources/js/services/api.js` is a configured Axios instance that injects the auth token, returns `response.data`, and centralizes error handling.

### Pinia Store: `resources/js/stores/usageStore.js`

- `fetchUsage(teamId)`: GET `/teams/{teamId}/usage` for member views.
- `fetchAdminTeamUsage(teamId, month)`: GET `/super-admin/teams/{teamId}/usage` with optional `?month=YYYY-MM`.
- `updateLimit(teamId, tokenLimitCost)`: PUT `/super-admin/teams/{teamId}/limit` with `{ token_limit_cost }`.

### Super Admin Pages

- List: `resources/js/pages/super-admin/Teams.vue`
  - Displays each team’s `usage_cost`, `limit_cost`, and `remaining_cost` for the current month.
  - No editing of limits on this page; click “View” to see details.

- Detail: `resources/js/pages/super-admin/Team.vue`
  - Month selector (`<input type="month">`) drives the `month` query param.
  - Displays totals and breakdowns for Responses and Chats: counts, tokens, and cost; total tokens and total cost; days until reset.
  - Allows editing the team’s `token_limit_cost` and saving via `updateLimit`.

## Operational Notes

- Migrations: run your migrations to add the new fields to `teams`.
- Null limits: A `null` `token_limit_cost` implies unlimited usage and skips enforcement.
- Enforcement timing: Calls to LLMs are blocked only when the accrued monthly `cost` already exceeds the limit. For stricter control, estimate and pass expected incremental cost.
- Data consistency: Ensure `cost` and `price` are set when creating `responses` and `chats`. If you add new LLM providers, update the token pricing config and the code that stores usage.

## Future Enhancements (Optional)

- Pre‑flight estimate: Pass `additionalCost` to `ensureWithinLimit` based on prompt/chat size.
- Alerts: Notify owners when remaining budget drops below thresholds.
- Hard vs. soft caps: Allow soft warnings and a hard stop at a higher cap.
